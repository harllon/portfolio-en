Projects

Color App

This application was developed in Kotlin and allows the user to view colors that can be chosen in three different ways. The first way consists of choosing the RGB components of the color through sliders, while the second way consists of writing the RGB components in hexadecimal. Finally, the third way is to choose the HSV components through "sliders". Viewing modes and the changes in color values ​​are synchronized using "Live Data". There is a help button in the app which opens a "dialog box" explaining the app itself. It is possible to login and register, for which Firebase was used, so that the user can save their favorite colors and view them later. The user can navigate between the possible ways to visualize the colors and the screen with the colors saved using tabs at the bottom of the screen, which were made using "viewPager". The saved colors are shown in the corresponding Tab, using the "Recycler View" allied to the "Room Database" that stores the colors saved by the user. Furthermore, the items in the "Recycler View" – in this case, the colors – are selectable, so that when you select a color, a "dialog box" opens, showing information about the saved color.

If you want to see details of code you can check the github of the project: Color App Github.

Sensor App

This application, developed in Java, allows the collection of data from several sensors available in cell phones, such as, the accelerometer and the gyroscope. The user can select multiple sensors from a list of options, as well as select some methods from the sensor data such as speed, altitude and motion verification. When you finish the selections and start the data collection, the chosen sensors are activated. This management, from knowing which sensors were selected to activating them, was done using Live Datas and the sensor library available in the Android documentation. At the end of the collection, if the user wants to save the data, they will be written in “.csv” files, one file for each sensor and method selected. To make this process viable, the Room Database was used in order to be able to store all the data that the user wants to save. The user can also share the files using other software, including e-mail, as the saved files will be automatically attached to the message body.

If you want to see details of code you can check the github of the project: Sensor App Github.

Synthetics Light Fields Dataset

This project was developed during my scientific initiation project, whose objective was to generate a dataset of synthetic light field videos that would become challenges for video compression algorithms. To produce this project, scenarios were created using Blender and Unity software, as well as a Blender plugin, responsible for creating a plenoptic camera. Scenarios were created by applying textures to objects in the scene and creating occlusion zones between objects, two techniques that make video compression difficult. The first makes compression "intra-frames" difficult - that is, within each "frame" of the video -, as this compression seeks similarities between image pixels and the texture creates a significant differentiation between pixels close of each other in the image. The second technique makes it difficult to compress "inter-frames", that is, between frames, as it is difficult to accurately predict where a given pixel will appear, especially if it enters an occlusion zone. At the end of the capture, the videos were visualized with a light fields video viewer that allowed exploring the two main properties of this video/image form, which are the change of focus after shooting and the change of point of view. This project had an article published in SIBGRAPI (Conference on Graphics, Patterns and Images), in 2020, and a presentation was made at the same congress.

If you want to see details of code you can check the github of the project: Synthetics Light Field Dataset.
